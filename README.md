# Solar PV Q&A Chatbot

A production-ready AI-powered Q&A chatbot for a Malaysian solar PV installation company. Users ask questions about solar energy and receive structured, branded HTML answers generated by an LLM and grounded in a curated knowledge base.

## Features

- **Streaming chat** — SSE-based real-time token streaming with cache-first lookup
- **Component library** — LLM outputs constrained HTML (tables, callouts, cards, steps, CTAs) styled by a shared design system
- **FAQ cache** — Pre-generated answers for common questions, with fuzzy matching (>70% word overlap)
- **Training feedback** — Good/Bad ratings and optional comments saved per AI response
- **Admin dashboard** — Browser-based knowledge editor with EasyMDE, token budget tracker, and FAQ cache management
- **Token budget tracking** — Per-file and total token counts with color-coded warnings (green < 25k, yellow 25k–35k, red > 35k)

## Quick Start

### 1. Install dependencies

```bash
npm install
```

### 2. Configure environment

Copy the example env file and fill in your LLM credentials:

```bash
cp .env.example .env
```

Required variables in `.env`:

| Variable | Description | Example |
|---|---|---|
| `LLM_API_KEY` | Your LLM API key | `sk-...` |
| `LLM_MODEL` | Model identifier | `mimo-v2.5-pro` |
| `LLM_BASE_URL` | API endpoint | `https://api.example.com/v1/messages` |
| `ADMIN_PASSWORD` | Admin dashboard password | `admin123` |
| `PORT` | Server port | `3000` |

### 3. Generate the FAQ cache

```bash
npm run generate-faq
```

This reads questions from `faq-cache/cache.json`, calls the LLM to generate polished HTML answers, and writes them back.

### 4. Run in development

```bash
npm run dev
```

This starts two processes concurrently:

- **Vite** on `http://localhost:5173` — serves the frontend with hot reload
- **Express** on `http://localhost:3000` — handles `/api/*` routes

Vite proxies `/api` requests to the Express server automatically.

### 5. Build for production

```bash
npm run build
```

This produces:

- `dist/` — Vite frontend bundle (HTML + JS + CSS)
- `dist/server/` — Compiled Express server

### 6. Run in production

```bash
npm start
```

The Express server serves both the API and the static frontend from `dist/`.

### Request Flow

```
User question
  → POST /api/chat { message: "..." }
    → Check FAQ cache (exact match, then fuzzy >70%)
      → Cache hit:  return JSON { cached: true, html: "..." }
      → Cache miss: build system prompt (knowledge + component spec + brand voice)
                    → Stream LLM response as SSE
                      → data: {"chunk":"token"}\n\n  (per token)
                      → data: [DONE]\n\n              (end of stream)
```

## Editing Knowledge

### Via the Admin Dashboard

1. Navigate to `http://localhost:5173/admin` (dev) or `http://localhost:3000/admin` (prod)
2. Log in with your `ADMIN_PASSWORD`
3. Browse files in the sidebar, click to edit
4. Save with Ctrl+S or the Save button
5. The manifest rebuilds automatically on save

### Direct File Edit

1. Edit any `.md` file under `knowledge/`
2. Run `npm run check-budget` to verify token counts
3. Restart the server (or it picks up changes on next request via lazy prompt caching)

### Adding New Knowledge Files

1. Create a new `.md` file in the appropriate category directory:
   ```
   knowledge/products/my-new-panel.md
   ```

2. The file should start with a clear title and contain structured information.

3. Run the manifest builder:
   ```bash
   npx tsx -e "import { buildManifest } from './server/knowledge/manifest.js'; buildManifest()"
   ```
   Or simply restart the server — it calls `buildManifest()` on startup.

4. Check the token budget:
   ```bash
   npm run check-budget
   ```

5. If the new file affects FAQ answers, regenerate the cache:
   ```bash
   npm run generate-faq
   ```

## Token Budget

The LLM has a context window limit. All knowledge files combined must fit within the system prompt budget.

| Zone | Tokens | Status |
|---|---|---|
| Green | < 25,000 | Healthy — room for more content |
| Yellow | 25,000–35,000 | Caution — consider trimming |
| Red | > 35,000 | Over budget — must trim files |

Check your current usage:

```bash
npm run check-budget
```

This prints a per-file breakdown and warns if you exceed the 35k budget.

The admin dashboard also shows a visual token budget bar under the **Token Budget** tab.

## FAQ Cache Management

The FAQ cache stores pre-generated HTML answers for common questions. When a user asks a question that matches a cache entry (exact or >70% word overlap), the server returns the cached answer instantly without calling the LLM.

## Internal Training Feedback

Every completed AI response includes Good/Bad controls and an optional comment field.
Submissions are upserted by response ID through `POST /api/feedback` and stored locally
in `training-data/feedback.json` with the question, answer HTML, invoice context (when
present), rating, comment, and timestamps.

### How it works

1. `faq-cache/cache.json` contains entries with `question`, `answer`, `topic`, and `stale` fields
2. On cache miss, the LLM generates an answer via streaming
3. When knowledge files change, affected cache entries are marked `stale`

### Regenerating the cache

```bash
# Regenerate all stale entries
npm run generate-faq -- --stale-only

# Regenerate everything
npm run generate-faq

# Regenerate a specific topic
npm run generate-faq -- --topic solar-pv
```

### Via the admin dashboard

The **FAQ Cache** tab shows all entries with their status (active/stale). Use the "Mark All Stale" button to flag entries for regeneration, then run `npm run generate-faq` from the CLI.

### Invoice-linked conversations

Open the chat with an invoice UID in the URL:

```text
http://localhost:5173/?invoice_uid=<INVOICE_UID>
```

For every question in that page, the server resolves the invoice's linked
package and adds the package name and `invoice_desc` to the AI context. Proxy
credentials remain on the server and are never exposed to browser code.
